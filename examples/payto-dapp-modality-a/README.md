# PayToDapp Modality A

Modality A is not part of the GlobalPayTo MVP.

The MVP path is Modality B:

- PayToDapps register supported route availability only.
- PayToDapps do not register wallet addresses, accounts, payment links, or chain-specific payment instructions.
- Dynamic destination details appear only inside the selected provider intent response.

Use [`../payto-dapp-modality-b`](../payto-dapp-modality-b) for the runnable
PayToDapp MVP example.
